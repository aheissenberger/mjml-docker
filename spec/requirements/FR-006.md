## Summary

The repository must ship a multi-stage `Dockerfile` that produces a minimal, production-ready
container image. The builder stage installs all dependencies and the runtime stage copies only
the production-necessary files, keeping the final image as small as possible. Both stages are
based on `node:25-bookworm-slim`.

## Acceptance Criteria

- `Dockerfile` exists at the repository root
- **Builder stage** (`FROM node:25-bookworm-slim AS builder`):
  - Copies `package.json` and the lock file, then runs `pnpm install --frozen-lockfile`
    to install all dependencies (including devDependencies needed at runtime for native
    type stripping)
  - Copies the full source tree
- **Runtime stage** (`FROM node:25-bookworm-slim`):
  - Enables corepack and activates the pnpm version pinned in `package.json`
  - Copies `package.json` and `pnpm-lock.yaml` from the builder stage
  - Runs `pnpm install --prod --frozen-lockfile` to install **production dependencies only**
    (excludes oxlint, oxfmt, typescript and other devDependencies)
  - Copies only `src/` from the builder stage
  - Runs as a non-root user (`node` user supplied by the base image)
  - Sets `NODE_ENV=production`
  - Requires `API_KEY` env var to be provided at container startup (per FR-005);
    containers started without it will exit immediately with a non-zero code
  - Exposes port 3000 (`EXPOSE 3000`)
  - Entrypoint: `node src/index.ts`
- `docker build .` succeeds without errors
- `docker run --rm -e API_KEY=test -p 3000:3000 <image>` starts the server and
  `GET /health` returns `{ "status": "ok" }`
- A `.dockerignore` file exists and excludes at minimum: `.git`, `node_modules`,
  `spec/`, `scripts/`, `test/`, `*.md`

## Verification

- Manual: `docker build -t mjml-docker .` succeeds
- Manual: container health-check passes (`curl http://localhost:3000/health`)
- Automated CI step (optional): `docker build` in the pipeline

## Notes

`node:25-bookworm-slim` ships Node 25 with Debian Bookworm. Because this project runs
TypeScript natively via Node's built-in type stripping (no compile step), the runtime
stage must have Node 25 available â€” there is no separate compiled JS artefact to copy.
The `pnpm` binary is activated in the runtime stage via `corepack enable && corepack
prepare pnpm@<version> --activate` using the version declared in `package.json`.
Related architectural decision: ADR-0004.
