## Summary

The service must start an HTTP server that listens on a configurable port (default 3000
via `PORT` environment variable) and respond to all HTTP requests with JSON content.

## Acceptance Criteria

- Server starts and listens on `PORT` env var when set, otherwise defaults to 3000
- Invalid `PORT` values (non-integer or outside 1-65535) fail fast at startup with a clear error
- Optional startup tuning variables (`RENDER_WORKERS`, `RENDER_QUEUE_SIZE`, `RENDER_TIMEOUT_MS`, `RATE_LIMIT_MAX_REQUESTS`, `RATE_LIMIT_WINDOW_MS`) are validated at startup and reject invalid values with clear errors
- Startup validation enforces upper bounds for runtime tuning values to prevent self-inflicted resource exhaustion
- On `SIGTERM`/`SIGINT`, the server begins graceful shutdown, stops accepting new requests, drains idle keep-alive connections, and exits within a bounded grace window
- All responses use `content-type: application/json` header
- Server runs directly from TypeScript source with `node src/index.ts` (no build step)

## Verification

- Smoke tested: `curl http://localhost:3000/` returns JSON
- Manual: `docker stop` (or Compose stop/down) exits without waiting for the full default timeout when the service is idle
- Automated: `pnpm test`

## Notes

Implemented with Node built-in `node:http`. No HTTP framework dependency.
Startup configuration parsing must be explicit and reject ambiguous numeric coercions.
Related architectural decision: ADR-0001.
