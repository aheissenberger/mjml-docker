## Summary

The repository must automatically publish the Docker image to GitHub Container Registry
(GHCR) through a manually triggered GitHub Actions workflow.

## Acceptance Criteria

- Workflow file exists at `.github/workflows/docker-publish.yml`
- Workflow triggers only via `workflow_dispatch`
- Package script `gh:docker-publish` exists and dispatches the workflow through `gh workflow run`
- Workflow authenticates to `ghcr.io` using `GITHUB_TOKEN`
- Workflow builds from the repository `Dockerfile` and pushes image tags to
  `ghcr.io/<owner>/<repo>`
- Workflow pushes both a version tag (release/tag value) and `latest`
- Workflow declares minimum permissions required for publish (`contents: read`, `packages: write`)

## Verification

- Static verification: workflow syntax is valid and manual trigger/permissions/publish steps are present
- Manual run: `pnpm gh:docker-publish -- <ref>` dispatches `workflow_dispatch` and validates GHCR image push in CI

## Notes

Version tag is resolved from the dispatched workflow ref (`github.ref_name`).
